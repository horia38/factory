# Factory Simulation - Manual Commands Guide

You can manually control the factory machines by publishing MQTT messages to their respective command topics.
You can do this using the Mosquitto CLI (`mosquitto_pub`) if installed, or by writing a quick python script.

## Sending Commands via Mosquitto CLI

Syntax: `mosquitto_pub -h localhost -t <topic> -m '<json_payload>'`

### 1. Dispenser (M1)
- **Refill Powder Hopper** (If it gets low or jams)
  `mosquitto_pub -h localhost -t factory/commands/machine1 -m '{"action": "refill"}'`

### 2. Granulator (M2)
- **Set Processing Speed** (Default ~800, affects viscosity and temperature)
  `mosquitto_pub -h localhost -t factory/commands/machine2 -m '{"action": "processing_speed_rpm", "value": 850}'`

### 3. Dryer (M3)
- **Set Target Heat** (Default 80.0°C, controls moisture reduction)
  `mosquitto_pub -h localhost -t factory/commands/machine3 -m '{"action": "target_heat_c", "value": 82.5}'`

### 4. Pill Press (M4)
- **Set Target RPM** (Default ~1000, higher speed = faster processing but higher vibration)
  `mosquitto_pub -h localhost -t factory/commands/machine4 -m '{"action": "speed_rpm", "value": 900}'`

### 5. QC Coater (M5)
- **Refill Coating Fluid** (If the coating fluid tank runs dry)
  `mosquitto_pub -h localhost -t factory/commands/machine5 -m '{"action": "refill_coating"}'`


## Sending Commands via Python

If you prefer using python, create a file (e.g. `send_cmd.py`) and run it:

```python
import paho.mqtt.client as mqtt
import json

client = mqtt.Client("ManualOverride")
client.connect("localhost", 1883)

# Example: Overriding Machine 3 heat
topic = "factory/commands/machine3"
payload = {"action": "target_heat_c", "value": 85.0}

client.publish(topic, json.dumps(payload))
print("Command sent!")
client.disconnect()
```

## Available Hard-Coded Failure Triggers

You can also run the existing trigger scripts to inject catastrophic failures into the system:
- `python trigger_m1_leak.py` - Drains the powder hopper instantly.
- `python trigger_m2_rpm_surge.py` - Forces M2 to 1200 RPM, causing severe friction heating.
- `python trigger_m4_vibration_jam.py` - Jams the M4 pill press, dropping speed to 200 RPM and causing pipeline backpressure.
